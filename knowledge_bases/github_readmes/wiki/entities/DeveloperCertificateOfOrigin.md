# Developer Certificate of Origin

## Overview
The Developer Certificate of Origin (DCO) version 1.1 is a legal statement used by the Linux kernel and many other open-source projects to establish that a contributor has the right to submit the work under the project's license. Every commit must include a Signed-off-by line, which acts as a certification of the DCO's terms.

## Key Facts
- **Purpose**: Legally certify that the contributor (or their employer) has the right to submit the code under the project's license
- **Signed-off-by format**: `Signed-off-by: Name <email@example.com>` — must be a real name (no pseudonyms)
- **DCO text**: "Developer's Certificate of Origin 1.1" — specifies that contributions are either created by the contributor or taken from public domain, and the contributor is legally empowered to submit them
- **Enforcement**: The Linux kernel enforces this for all submissions; mailing-list bots and maintainers check for it
- **AI-specific mandate**: The Linux kernel explicitly requires LLMs and AI coding assistants to comply with the DCO before contributing, as stated in Documentation/process/coding-assistants.rst
- **Other projects**: Used beyond Linux — many large open-source projects (e.g., Docker, Kubernetes) adopt similar or identical DCO language
